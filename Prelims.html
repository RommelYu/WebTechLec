    The World Wide Web (WWW) or the web is an information space where documents and other web resources are identified by Uniform Source Locator's (URLs), interlinked by hypertext links, and can be accessed via the Internet.

    The World Wide Web has been central to the development of the Information age and is the primary tool of billions of people use to interact on the Internet.

    What is HTTP based on?

    HTTP runs on top of TCP/IP using port 80 by default, TCP port 443 is for HTTPS. It is based on client server architecture.

    HTTP is based on a client-server architecture, wherein its entities are clients, with their user-agents such as web browsers, web crawlers and other end user tools and applications, and as with servers, such as origin servers, proxy agents (http://10.10.33.254), gateways and tunnels.

    HTTP uses a request-response standard protocol, wherein the client sends an HTTP request message to the server, after which it processes the request and replies with an HTTP response message.

    HTTP is a stateless communications protocol (pull protocol, also known as polling, which refreshes at regular intervals, push is later introduced by HTTP 2.0). Servers do not keep information about clients in between requests.

    HTTP provides support for other functionalities, such as...

    • cache control

    • content media type specification (MIME)

    • language and character set specification

    • content/transfer codings

    • content negotiation

    • client-server protocol negotiations

    • persistent connections

    • request pipelining

    • authentication/authorization

    HTTP Resource Addressing

    HTTP resources are identified using URI, or , more, specifically, HTTP URL

    HTTP Request Message

    • Request Line (CRLF-terminated line of 3-space separated values)

      • Line Part Description
      Methods It indicates the method to be performed on the resource identified by the Request-URI. Case-sensitive.
      Request URI The Request-URI is a Uniform Resource Identifier and identifies the resource upon which to apply the request.

    • Message Headers (general requests and/or entity headers)

      • HTTP 1.1 requires at least the Host request header to be provided

    • Empty Line (CRLF)

    • Message Body, aka. payload (optional)


    HTTP Request Methods

    Method Description
    OPTIONS Represets a request for information about the communication options availbale on the request/responsse chian identified by the Request-URI.
    GET Transfers a current selected representation of the resouce identified by the request URI: the retrieved resource is returned in the message body of the response as an entity
    HEAD The HEAD method is identical to GET except that the server MUST NOT return a message-body in the response.
    POST Used to request that the origin server accept the entity enclosed int he request as a new subordinate of the resource identified by the Request-URI in the Request-Line.
    PUT Requests that the enclosed entity be stored under thge supplied Request-URI.
    DELETE Requests that the origin server delete the resource identified by the Request-URI.
    TRACE Used to invoke a remote, application-layer-loop-back of the request message.
    CONNECT Requests the establishment of a tunnel to the destination origin server, and if successful thereafter restrict its behaiour to blind-forwarding of packets in both directions, until the tunnel is closed.
