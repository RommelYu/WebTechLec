HTTP Fundamentals

  • HTTP typically runs on the top of TCP/IP, using TCP port 80 by default, or TCP port 443 for HTTPs (HTTP over SSl/TLS)
  • HTTP is based on client – server architecture
    •    o Servers:
       • Origin Servers
       • Proxy servers, gateways, tunnels
       o Clients, a.k.a. user agents(UA):
       • Web browsers, web crawlers/spiders, other end user tools and applications
  • HTTP uses a request – response standard protocol
    •    o The client sends an HTTP request message to the server
       o The server processes the request and replies with an HTTP response message.
  • HTTP is a stateless communications protocol
  • HTTP provides support for other functionalities such as:
    •    o Cache
       o Content media type (MIME) specification
       o Language and character set specification
