HTTP Request Method

  • Standard Method
    •    o Get - transfer a current selected
       o Head - identical to get, accept that the server does not send a message
       o Post - request that the target resource perform resource - specific processing of presentation enclosed in the message body of the request
       • Query - typically provide as key = value pairs, with ampersand (&)separators between key/value pairs
       • Path - to resource (resolved relative to the document root on the server)
       • Fragment Identifier
       o Put - create or replace the state of the target source with the state defined by the presentation enclosed in the request message payload
       o Delete - remove the association between the target resources and its current functionality
       o Options - allows to query about the server on what to do, request information about the communication
       o Trace - request a remote, application – level loop –back of the request message (ie; request the recipient to echo back to the client the request message). Typically used for testing/diagnostics of the request/response chain
       o Connect - requests the establishment of a tunnel to the destination origin server, and if successful, there after restrict its behavior to blind – forwarding of packets, in both directions, until tunnel is closed.
  • Method Properties
    •    o Safe Method - client does not request, and does not expect, any state change on the origin server as a result of applying the method to a target source. GET,HEAD, OPTIONS, TRACE.
       o Idempotent Method – intended effect on the server of multiple identical requests with the methods is the same as the effect of a single such request. GET, HEAD, OPTIONS, TRACE, PUT DELETE
       o Cacheable Method – indicates that the response to a method is allowed to be stored for future use. GET, HEAD, POST
